CLARIFICATION SUR LES “LIENS GITHUB”
JE M’EXCUSE POUR LA CONFUSION : JUSQU’À PRÉSENT, JE N’AI PAS FOURNI DE VÉRITABLES URL GITHUB POUR HÉBERGER TES ASSETS. DANS MES TABLEAUX, LES MENTIONS “LINKEDIN / GITHUB / PDF” SONT DES SUGGESTIONS DE SUPPORTS OÙ DÉPLOYER TES FICHIERS, PAS DES LIENS ACTIFS.

Héberger ton kit sur GitHub en trois étapes
- Créer un dépôt
- Connecte-toi à github.com
- Clique sur New repository
- Nomme-le par exemple EA-Branding
- Choisis Public (ou Private) et valide
- Ajouter tes fichiers
- Dans ton dépôt, clique sur Add file > Upload files
- Glisse-dépose les fichiers :
- EA_sublime_bleu_original.svg
- EA_constellation_vapeur.png
- etc.
- Commit ta mise à jour
- Récupérer les liens
- Ouvre un fichier (ex. EA_sublime_bleu_original.svg) dans GitHub
- Clique sur Raw
- Copie l’URL (format : https://raw.githubusercontent.com/Utilisateur/EA-Branding/main/EA_sublime_bleu_original.svg)
- Utilise ce lien dans tes pages HTML, CSS ou ta documentation

Exemple d’utilisation en CSS
.logo {
  width: 200px;
  height: auto;
  background: url('https://raw.githubusercontent.com/EricDev/EA-Branding/main/EA_sublime_bleu_original.svg') no-repeat center;
}
